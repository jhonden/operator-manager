# 算子包批量添加算子功能 - 需求设计文档

**文档版本**: v1.1
**创建日期**: 2026-02-23
**状态**: 设计中

---

## 1. 需求概述

### 1.1 需求背景

当前算子包详情页面只支持一次添加一个算子，用户需要多次打开弹窗、多次选择算子、多次确认，操作流程繁琐。当需要将多个算子添加到算子包时，用户体验较差，效率低下。

### 1.2 需求目标

优化算子包添加算子的易用性，支持批量添加多个算子，提升用户操作效率和体验。

### 1.3 原始需求记录

用户反馈：当前只支持一次添加一个算子，算子比较多时就比较麻烦，希望支持批量添加。

## 2. 功能需求

### 2.1 功能描述

在算子包详情页面的"添加算子"弹窗中，支持同时选择多个算子并批量添加到算子包中。

### 2.2 用户故事

**场景一：批量添加并行执行的算子**

作为系统用户，我希望能够一次性选择多个需要并行执行的算子，统一设置执行顺序为 1，然后一次性添加到算子包中，避免多次操作的繁琐流程。

**场景二：批量添加相同业务流程的算子**

作为系统用户，我希望能够搜索和筛选算子列表，然后勾选同一业务场景下的多个算子，批量添加到算子包中，提升工作效率。

### 2.3 功能清单

#### 前端功能

1. **多选算子列表**
   - 弹窗中间区域显示可用算子列表
   - 支持复选框多选
   - 支持关键词搜索算子名称
   - 支持按编程语言筛选（Java/Groovy）
   - 显示算子基本信息（名称、语言、描述）
   - 列表支持滚动（scroll={{ y: 300 }}）

2. **统一执行顺序设置**
   - 弹窗顶部提供统一的执行顺序输入框，始终可见
   - 所有选中的算子使用相同的 orderIndex
   - 默认值为 1
   - 验证：orderIndex 必须 >= 1

3. **已选数量提示**
   - 弹窗顶部显示"已选择 N 个算子"
   - 选择算子时实时更新数量

4. **添加确认**
   - 点击"确认添加"按钮一次性添加所有选中的算子
   - 确认按钮固定在弹窗底部（Modal Footer），始终可见
   - 添加时显示加载状态
   - 添加成功后关闭弹窗并刷新算子列表
   - 失败时显示错误提示

5. **交互优化**
   - 选择算子时实时更新已选数量
   - 执行顺序和已选数量在弹窗顶部固定，不会被列表淹没
   - 确认按钮在弹窗底部固定，列表滚动时始终可见

#### 后端功能

1. **批量添加算子 API**
   - 接受算子 ID 列表和统一的执行顺序
   - 循环添加每个算子到算子包
   - 自动同步每个算子的公共库依赖
   - 返回添加结果（成功/失败的算子列表）

2. **错误处理**
   - 部分算子添加失败时，返回失败的算子信息
   - 前端显示哪些算子添加成功，哪些失败

## 3. 非功能需求

### 3.1 性能要求

- 批量添加 10 个算子的响应时间不超过 3 秒
- 算子列表加载时间不超过 1 秒

### 3.2 安全要求

- 需要用户登录认证
- 验证算子 ID 的有效性
- 验证用户是否有操作该算子包的权限

### 3.3 可用性要求

- 弹窗界面简洁，操作流程清晰
- 已选择的算子有明显的视觉标识
- 执行顺序和确认按钮始终可见
- 支持键盘操作（Tab 切换、Enter 提交）

## 4. 技术方案

### 4.1 整体设计

采用方案一：多选弹窗 + 统一顺序设置（执行顺序置顶，确认按钮固定底部）。

**交互流程**：
1. 用户点击"添加算子"按钮
2. 弹窗显示，顶部显示执行顺序输入框和已选数量提示
3. 用户搜索/筛选算子
4. 用户勾选多个算子（复选框），已选数量实时更新
5. 用户在顶部统一设置执行顺序
6. 点击底部"确认添加"按钮调用批量添加 API
7. 后端循环添加每个算子，自动同步公共库
8. 前端刷新算子列表并关闭弹窗

**界面布局**：
```
┌─────────────────────────────────────────┐
│  添加算子到包                    ✕  │
├─────────────────────────────────────────┤
│  ┌───────────────────────────────────┐  │

│  └───────────────────────────────────┘  │
│  已选择 3 个算子                        │  ← 顶部固定区域
│                                         │

│  │ ☐  算子名称     语言    描述    │  │
```

**布局说明**：
- **顶部固定区域**：执行顺序输入框 + 已选数量提示，始终可见
- **中间可滚动区域**：搜索框、筛选框、算子列表（列表过长时可滚动）
- **底部固定区域**：取消和确认按钮，始终可见

### 4.2 数据库设计

无需数据库结构变更，使用现有表结构：

- `package_operators` - 算子包-算子关联表
- `package_common_libraries` - 算子包-公共库关联表（自动同步）

### 4.3 接口设计

#### 新增 API：批量添加算子到算子包

**接口路径**: `POST /v1/packages/{id}/operators/batch`

**请求参数**:
```json
{
  "operatorIds": [1, 2, 3],
  "orderIndex": 1,
  "enabled": true
}
```

**请求参数说明**:
| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| operatorIds | number[] | 是 | 算子 ID 列表 |
| orderIndex | number | 是 | 统一执行顺序（>= 1）|
| enabled | boolean | 否 | 是否启用，默认 true |

**成功响应**:
```json
{
  "success": true,
  "message": "成功添加 3 个算子到包",
  "data": {
    "total": 3,
    "successCount": 3,
    "failedCount": 0,
    "failedOperators": []
  }
}
```

**失败响应**:
```json
{
  "success": false,
  "message": "部分算子添加失败",
  "data": {
    "total": 3,
    "successCount": 2,
    "failedCount": 1,
    "failedOperators": [
      {
        "operatorId": 3,
        "operatorName": "MathUtil",
        "reason": "该算子已在包中"
      }
    ]
  }
}
```

**错误处理**:
- 算子 ID 不存在：返回 404
- 算子已存在：跳过该算子，记录到 failedOperators
- 权限不足：返回 403
- 参数验证失败：返回 400

#### 现有 API 保持不变

- `POST /v1/packages/{id}/operators` - 单个添加算子（保留，用于向后兼容）
- 其他算子包相关 API 不变

```typescript
  return availableOperators.filter(op => {
    const matchKeyword = !searchKeyword ||
      op.name.toLowerCase().includes(searchKeyword.toLowerCase());

    return matchKeyword && matchLanguage;
```

#### 4.4.2 API 调用

```typescript
const handleBatchAddOperators = async () => {
  if (!id || selectedOperatorIds.length === 0) {
    message.warning('请选择至少一个算子');
    return;
  }

  setAddOperatorLoading(true);
  try {
    const values = addForm.getFieldsValue();
    const response = await packageApi.batchAddOperators(Number(id), {
      operatorIds: selectedOperatorIds,
      orderIndex: values.orderIndex || 1,
      enabled: true,
    });

    if (response.data) {
      const { successCount, failedCount, failedOperators } = response.data;
      if (failedCount > 0) {
        message.warning(
          `成功添加 ${successCount} 个算子，${failedCount} 个失败`
        );
        // 显示失败详情
        console.log('[Package Page] Failed operators:', failedOperators);
      } else {
        message.success(`成功添加 ${successCount} 个算子到包`);
      }
    }

    handleCloseAddModal();
    fetchPackage();
  } catch (error: any) {
    console.error('[Package Page] Batch add operators failed:', error);
    message.error(error.response?.data?.error || error.message || '批量添加算子失败');
  } finally {
    setAddOperatorLoading(false);
  }
};
```

### 5.3 风险评估

| 风险 | 影响 | 概率 | 应对措施 |
|-----|------|------|---------|
| 批量添加性能问题 | 高 | 中 | 设置批量数量上限（如 50 个），超时分批处理 |
| 公共库同步失败 | 中 | 低 | 捕获异常，记录失败信息，不影响算子添加 |
| 部分算子添加失败 | 中 | 中 | 返回详细的失败信息，前端显示哪些算子失败 |
| 前端弹窗布局适配 | 低 | 低 | 测试不同屏幕尺寸，确保按钮始终可见 |
